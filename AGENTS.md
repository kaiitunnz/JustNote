# AGENTS.md

Guidance for coding agents working on JustNote. Agent-specific entry files (e.g. `CLAUDE.md`)
import from here.

## What JustNote is

A macOS **accessory** app (`LSUIElement`, no Dock icon at rest, no menu-bar icon) for plain-text
notes. The UI is SwiftUI (`MenuView`) hosted in an AppKit-managed floating **`NSPanel`** summoned by
a global shortcut (default ⌥.). Settings is a separate AppKit `NSWindow`. Notes are `.txt` files
autosaved to `~/Library/Application Support/JustNote/Notes`, with app state in `state.json` beside
them.

Targets **macOS 27+**, Swift 5.

## Build, test, run

`project.yml` is the source of truth; the `.xcodeproj` is generated by **XcodeGen** and git-ignored.

```sh
xcodegen generate          # after adding/removing files (sources are globbed by directory)
xcodebuild -project JustNote.xcodeproj -scheme JustNote -configuration Debug -derivedDataPath build build
xcodebuild test  -project JustNote.xcodeproj -scheme JustNote -configuration Debug -derivedDataPath build -destination 'platform=macOS'
```

The XCTest suite is **model-only** (injects a temp store) — it covers `NoteStore`/`AppModel`, not
any window/panel/keyboard behavior. **GUI behavior must be verified by driving the running app**; see
the `justnote-verify` skill in `.claude/skills/justnote-verify/`, which has the build/launch recipe
and a CGEvent-based GUI-drive kit. Note that driving the app operates on the **real** user note store.

## Architecture

- **`PanelController` / `FloatingPanel`** — owns the summoned panel: summon/toggle/dismiss,
  positioning, frame persistence. `AppDelegate` registers the global shortcut to `toggle()`.
- **`AppDelegate` / `JustNoteApp`** — accessory lifecycle. Stays `.accessory`; flips to `.regular`
  only while Settings is open. Every **cold launch opens Settings** (the pointer-driven entry point,
  since there's no menu-bar icon); `applicationShouldHandleReopen` opens Settings on Dock click.
- **`MenuView`** — the entire panel UI (sidebar + editor + header + footer). Shared, chrome-agnostic;
  the hosting window supplies material/rounded chrome.
- **`NoteStore` / `AppModel` / `Note`** — storage and note model. **`AppModel.isQuitting`** gates the
  soft-quit: ⌘Q hides to the background; only the panel's Quit button (or system logout) terminates.

## AppKit window facts (learned empirically — don't relitigate)

The panel is a **chromeless titled window**: `styleMask = [.titled, .fullSizeContentView, .resizable]`
with `titleVisibility = .hidden`, `titlebarAppearsTransparent = true`, and the standard window buttons
hidden. Rationale, each a real dead end tried and rejected:

- **A borderless `NSPanel` cannot become key** without overriding `canBecomeKey`/`canBecomeMain`, and
  **borderless + `.resizable` gives no interactive edge-resize.** Titled + `fullSizeContentView` is
  what makes it natively resizable while looking chromeless.
- **`fullSizeContentView` gives the `NSHostingView` a top titlebar safe-area inset** — set
  `hostingView.safeAreaRegions = []` or `MenuView` renders with an empty strip above the header.
- **Do NOT set `isMovableByWindowBackground = true`.** It lets background drags move the window, which
  **hijacks note-row drag-and-drop** (a shipped feature). Dragging is provided by the transparent
  **titlebar strip** instead; header buttons sit below it and still receive clicks.
- **`windowShouldZoom` returns `false`** — `.resizable` makes the panel zoomable, and a titlebar
  double-click would otherwise balloon it and persist that frame.
- **Frame persistence** is `setFrameAutosaveName` (auto-saves on move/resize) + `setFrameUsingName`
  restored on first summon; off-screen frames recenter.
- **Dismissal** is standard window behavior: the toggle shortcut, Escape (a local `keyDown` monitor,
  because `cancelOperation` never reaches the panel while the editor's `NSTextView` is first responder;
  it yields to IME marked text), and resign-key. The resign-key handler is guarded against closing
  behind app-modal alerts (`withDismissSuspended`), when another of our windows takes key/main, when
  the panel has regained key, and during a short grace window after summoning over a full-screen space.
- **Never anchor the panel to an `NSStatusItem`.** That coupling was the original keyboard bug: macOS
  routed the first Space/Return to the menu-bar item and synthesized a click that dismissed the panel.
  A standalone key window has no such anchor.

## Dependencies (pinned deliberately)

- **MarkdownView** — pinned to a `main` **commit**; the 2.7.0 release fails to compile against the
  macOS 27 SDK (`Document` ambiguity).
- **KeyboardShortcuts** — `exactVersion` 1.9.4. Backed by Carbon `RegisterEventHotKey`, so the global
  shortcut needs **no Accessibility/Input-Monitoring prompt**. Pinned exactly because the `.xcodeproj`
  (and its `Package.resolved`) is git-ignored, so a range would let checkouts drift.

## Conventions

- **Commits:** imperative subject; DCO sign-off required (`git commit -s`). Keep the body for the
  non-obvious *why*.
- **Branches / PRs:** `feat/…` (or `chore/…`, `docs/…`) branches; never commit to `main`. PRs are
  **squash-merged**. Match the existing PR body shape (`## Summary` / `## Changes` / `## Test Plan`,
  and state which GUI checks were vs. weren't run).
- **Comments:** sparse — explain *why*, not *what*; names carry the *what*.
